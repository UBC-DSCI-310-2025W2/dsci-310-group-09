# Contributing

We welcome all contributions ot this project! Thank you for your interest in improving our work.

## 1. How to Contribute

There are many ways to contribute, including:

- Reporting bugs
- Suggesting new features or improvements
- Improving documentation
- Enhancing code or analysis
- Adding tests or examples

## 2. Reporting Issues

When opening an issue, please include:

- A clear and descriptive title
- Steps to reproduce the problem
- Expected and actual behaviour
- Relevant screenshots or error messages
- Your system and software environment (if applicable)

## 3. Submitting Changes

If you'd like to contribute a feature or bug fix:

1. Fork the repository.
2. Create a new branch for your work.
3. Make your changes.
4. Commit your changes with clear and descriptive message.
5. Push your branch to your fork.
6. Submit a pull request to our main repository.

We aim to review pull requests within **7 days**.

## 4. Development Guidelines

To help maintain quality and reproducibility:

- Follow clean and readable coding practices.
- Write clear documentation and comments.
- Avoid hard-coded file paths.
- Ensure your work is reproducible.
- Update documentation if your changes affect usage.
- Test your changes before submitting.

## 5. Code Style

Please ensure your code:

- Is consistent with the existing project style.
- Uses meaningful variable and function names.
- Includes comments where needed.
- Follows reproducible and ethical data science practices.

## 6. Code of Conduct

All contributors must abite by our **Code of Conduct**. 
By participating, you agree to uphold a respective and inclusive community.

## 7. Questions

If you have any questions about contributing, please open an issue and we will be happy to help.

Thank you for helping make this project better!
